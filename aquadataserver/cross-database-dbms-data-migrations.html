Cross-DBMS Data Migrations
Aqua Data Server

Aqua Data Server functions as a powerful and versatile tool for migrating data between databases of different vendors (e.g. from MySQL to IBM DB2, or vice-versa). With customized AquaScripts, data can be extracted from a source database, modified to match the format of the destination, and updated in the destination database. The migration process may be accomplished without intermediate storage of the data in the file system. Aqua Data Server works when you need it; the built-in scheduler can execute data migration scripts during specified time intervals to match business downtime, or to control server load in production environments. This allows you to perform data migrations on live, production systems. Any migration logs or warnings may be emailed to database administrators using the Aqua Data Server reliable email queue.



Use these Aqua Data Server packages:

  • aquafold.openapi.rdbms and aquafold.openapi.rdbms.schema to access and manage databases of different vendors;
  • aquafold.openapi.math to implement data processing algorithms;
  • aquafold.openapi.type for manipulating various types of data.
